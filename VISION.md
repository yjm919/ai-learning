# AI 知识库 — 项目愿景文档

## 使命

让每一个 AI 从业者都不再被信息淹没——用 Agent 系统替代人工情报劳动，把全球 AI 动态变成可以检索、可以推送、可以辅助决策的知识资产。

## 我们解决什么问题

AI 领域每天产生数百篇论文、上千个开源项目、无数社区讨论。技术人面临三重困境：

- **信息过载**：量太大，看不过来，真正重要的被噪音淹没。
- **碎片化**：GitHub、HN、arXiv、Twitter、公众号……信息散落在数十个渠道，缺少统一入口。
- **加工成本高**：即便看到了，还要花时间读、分类、判断价值、与已有知识关联。

结果是大部分人要么放弃跟踪、要么依赖某个人的私人笔记——系统性的知识管理几乎不存在。

## 目标用户

| 用户画像 | 核心诉求 |
|---------|---------|
| AI 工程师 / 研究员 | 不遗漏领域内关键进展，快速判断"这东西值得试吗" |
| 技术管理者 / CTO | 宏观把握技术趋势，辅助技术选型与团队方向决策 |
| AI 创业者 / 投资人 | 发现早期信号与新机会，追踪竞品与范式转移 |
| 技术内容创作者 | 获取选题素材，理解社区情绪与热点 |

## 产品定位

**一个"AI 领域的 Bloomberg Terminal 轻量版"**——不是通用 RSS 阅读器，不是通用搜索引擎，而是专为 AI 技术情报打造的垂直知识管线。核心词：**自动采集 → Agent 分析 → 结构化 → 可检索 → 可推送**。

## 核心功能

```
┌───────────┐    ┌───────────┐    ┌───────────┐    ┌───────────┐
│  采集层    │ → │  分析层    │ → │  整理层    │ → │  分发层    │
│ Collector │    │ Analyst   │    │ Organizer │    │ Publisher │
└───────────┘    └───────────┘    └───────────┘    └───────────┘
     ↓                ↓               ↓               ↓
 GitHub Trending   摘要 + 标签    去重 + 审核     Telegram / 飞书
 Hacker News       评分 + 分类    批量发布       邮件 / Slack
 arXiv papers      关联已有条目   质量控制        可扩展渠道
```

1. **多源采集**：定时从 GitHub Trending、Hacker News、arXiv 抓取 AI/LLM/Agent 相关内容。
2. **Agent 协作分析**：由 3 个 AI Agent 分工协作——采集 Agent 拉取原始数据，分析 Agent 生成摘要、打标签、评分、去重，整理 Agent 审核后批量发布。
3. **结构化落库**：分析结果统一存储为 JSON，支持按标签、来源、评分、时间等多维度检索。
4. **多渠道推送**：通过 Telegram Bot、飞书 Bot 实时推送精选条目；未来扩展邮件、Slack、自定义 Webhook。
5. **知识检索**（规划中）：向量化存储 + RAG 语义搜索，让历史知识可以被提问调用。

## 技术理念

- **Agent 优先**：不是把 AI 当 API 调，而是让多个 Agent 组成管线，各自负责一个明确环节，可替换、可观测、可优化。
- **模型可插拔**：通过 OpenCode 的统一适配层屏蔽模型差异，国产模型 / OpenAI / Claude 无缝切换。
- **数据主权**：所有知识数据本地落盘，不依赖外部 SaaS，用户可以完全掌控自己的知识库。
- **渐进式增强**：先保证"跑通"（采集+分析+推送），再追求"好用"（搜索+RAG+个性化）。

## 成功指标

| 指标 | 定义 | 目标（首版） |
|------|------|------------|
| 采集覆盖率 | 目标源的有价值条目被采集的比例 | > 95% |
| 分析准确率 | 摘要无事实错误、标签命中主题 | > 90%（人工抽检） |
| 去重率 | 跨源重复条目被正确识别的比例 | > 85% |
| 推送延迟 | 从信息首次出现到推送的时间 | < 6 小时 |
| 用户留存 | 订阅 30 天后仍在接收推送的活跃用户比例 | > 60% |

## 不做的事（明确边界）

- **不做实时新闻**：定时采集而非实时流，追求准确性而非速度。
- **不做社交功能**：不建社区、不搞评论、不搞点赞。
- **不做全领域覆盖**：聚焦 AI / LLM / Agent 领域，不扩展到前端、运维、数据库等。
- **不做内容生成**：只做摘要和分类，不写原创文章、不生成教程。
- **不替代论文精读工具**：提供概览和索引，不做逐段翻译或深度解读。

## 愿景

6 个月后的理想场景：用户每天早上打开飞书/Telegram，一条消息告诉自己过去 24 小时 AI 领域发生了什么真正重要的事——每个条目都有摘要、有评分、有出处链接。不需要刷 GitHub Trending 页面，不需要逐篇翻 arXiv，不需要在 HN 评论区考古。知识自己会找到对的人。
